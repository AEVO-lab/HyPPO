Normally, you can compile OCR with the usual> ./configure> make> make installThis should work fine. If it doesn't, another option is to compile with Qt (the program was developed using QtCreator, but there are no dependencies to any of the Qt libraries).Whichever environment is used, the user must ensure that the equivalent to the following Qtcompiler directives are handled : QMAKE_CXXFLAGS += -std=c++0xThe project requires the use of c++0x (for the unordered_set and the unordered_map classes).If Qt is installed, the simplest to to get into the folder, and enter the two commands> qmake OCR.pro "CONFIG+=RELEASE"> make
